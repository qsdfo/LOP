Description

This is a MIDI database of 223 pairs of piano scores and corresponding orchestrations performed by famous composers. The figure of the right-hand side represents the hierarchy of the database and general statistics are given in the table. $$x = {-b \pm \sqrt{b^2-4ac} \over 2a}.$$

The dataset is split between train, validation and test sets of files that represent approximately 80%, 10% and 10% of the full set. The details of the split we used is given in text files constructed as the name of the folders and the name of the subset. For instance, the files from the liszt_classical_archives used for the training step are listed in the liszt_classical_archives_train.txt file.

Warning : We suggest not to use the IMSLP database which contains a lot of non-matching pairs. Cleaning will be done as soon as possible.

Data representation in LOP

We used a simple piano-roll representation to process the orchestral and piano scores in LOP. A piano-roll is a matrix whose rows represent pitches and columns represent a time frame depending on the time quantization. A pitch p at time t played with an intensity i is represented by $pr(p,t) = i$, $0$ being a note off. This definition is extended to an orchestra by simply concatenating the piano-rolls of every instruments along the pitch dimension.
